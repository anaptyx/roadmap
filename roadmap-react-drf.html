Roadmap: Full‑stack React + DRF (от нуля до джуна/мидла)

Этот документ — подробный учебный план с упором на фундамент каждого направления. По каждому блоку есть: цели, что выучить (ядро), практические шаги, артефакты в портфолио и критерии готовности. Отмечайте галочками чек‑боксы по мере продвижения.

Как работать с планом

  • Идите слоями: основа → практика → рефакторинг → тесты.
  • Каждую неделю создавайте артефакт: коммит, мини‑проект, заметку, тесты.
  • Время ориентировочно дано для режима 15–20 ч/нед. Ускорять можно за счёт мини‑проектов.

0) Подготовка окружения и навыки разработчика (0–1 неделя)

Цели: уверенно работать с инструментами и контролировать среду.

Что выучить (фундамент):

  • Командная строка: навигация, alias’ы, управление процессами, curl, grep, sed, ssh.
  • Git: clone, branch, commit, rebase, merge, stash, cherry-pick, tag, PR/Code Review.
  • VS Code: multi‑cursor, debug, tasks, launch configs, snippets.
  • Node LTS, npm/yarn/pnpm; Python 3.12+, venv/poetry.
  • Форматирование и линтинг: Prettier/ESLint (JS/TS), Black/isort/flake8 (Python), EditorConfig.
  • Инструменты API: Postman/Insomnia; JSON Schema/OpenAPI (на уровне чтения).

Практика:

  • Настройте репозиторий starter с шаблонами: lint/format hooks (Husky/ruff pre-commit), CI на GitHub Actions.
  • Соберите dotfiles или VS Code Settings Sync.

Критерии готовности:

1) Веб‑фундамент: HTML/CSS/Доступность + Сеть/HTTP + Базовый JavaScript (2–3 недели)

Цели: понимать, как строится страница, как она грузится и общается с сервером.

1.1 HTML/CSS/Accessibility

Что выучить:

  • Семантика HTML5: header/main/nav/section/article/aside/footer, формы (label, fieldset, aria-*).
  • Макеты: Flexbox, Grid, responsive, fluid typography, мобильное меню.
  • CSS: каскад/специфичность, переменные (custom properties), BEM/utility‑first, анимации (transition/transform).
  • A11y: фокус‑менеджмент, контраст, клавиатурная навигация, aria-live.

Практика:

  • Лендинг + форма обратной связи (валидация на JS) + тёмная тема.

Критерии:

1.2 Сеть и HTTP

Что выучить:

  • Жизненный цикл запроса: DNS → TCP/TLS → HTTP/2.
  • Методы/статусы/заголовки, кеширование (ETag/Cache‑Control), CORS, cookies, SameSite.
  • REST принципы, idempotency, пагинация, фильтры, ошибки.

Практика:

  • Снимите запросы DevTools, разберите кеш/ререндер.

Критерии:

1.3 JavaScript (современный)

Что выучить:

  • Типы/объекты/массивы, замыкания, прототипы, модули, async/await, промисы, генераторы, итераторы.
  • Работа с DOM/Events, fetch/AbortController, таймеры, дебаунс/троттлинг.
  • Ошибки: try/catch, Error, стек, логирование.

Практика:

  • To‑Do без библиотек: добавление/редактирование/фильтры/персист в localStorage.

Критерии:

2) React‑фундамент (4–6 недель)

Цели: уметь собирать UI‑сцены, управлять состоянием, работать с данными.

Что выучить:

  • Архитектура: компонентный подход, пропсы, композиция, lifting state, контейнер/презентационный компонент.
  • Хуки: useState, useEffect (зависимости и гонки), useMemo, useCallback, useRef, кастомные хуки.
  • Роутинг: React Router (layout routes, protected routes, lazy loading, loaders/actions при необходимости).
  • Формы: React Hook Form + Zod/Yup (схемы, резолвер, формы многошаговые).
  • Состояние: Redux Toolkit (slices, RTK Query) или Zustand/Jotai — понимать trade‑offs.
  • Data fetching: паттерны (stale‑while‑revalidate, optimistic updates), кеш, отмена запросов.
  • UI/стили: Tailwind CSS (utility‑first, responsive, dark), Headless UI/shadcn (по необходимости).
  • Тестирование: Jest + React Testing Library (юнит, интеграции компонент, моки API), coverage.
  • Производительность: мемоизация, ключи, виртуализация списков, React.Suspense (базово).
  • Бандлинг: Vite, code‑splitting, env‑переменные.
  • (Рекомендовано) TypeScript: типы пропсов, дженерики для хуков, утилиты Partial/Record/Pick, типизация RTK Query.

Практика (проект):

  • Task Dashboard SPA: аутентификация mock, CRUD досок/колонок/карт, поиск/фильтры, drag‑and‑drop, сохранение предпочтений.

Артефакты:

  • README с архитектурой, скриншоты, E2E‑демо (Vercel/Netlify).

Критерии:

3) Python‑фундамент (1–2 недели)

Цели: уверенно писать чистый, типизированный код.

Что выучить:

  • Язык: типы, коллекции, срезы, списковые включения, *args/**kwargs, контекстные менеджеры, исключения.
  • Файловый ввод/вывод, сериализация JSON, dataclasses.
  • ООП: наследование, композиция, протоколы/abc.
  • Типизация: typing, mypy, pydantic (как инструмент валидации — концептуально).
  • Управление зависимостями: venv/poetry.
  • Тесты: pytest, фикстуры, параметризация, coverage.

Практика:

  • Утилита импорта/экспорта JSON↔CSV, покрытая тестами.

Критерии:

4) Django‑фундамент (2–3 недели)

Цели: модель‑ориентированная разработка и уверенная работа с ORM.

Что выучить:

  • Проект/приложения, настройки (env‑переменные, django-environ), миграции.
  • Модели: поля/сигналы, Meta, индексы, уникальные ограничения, связи (FK/M2M/OneToOne), валидаторы.
  • ORM: фильтрация/агрегации/аннотации, select_related/prefetch_related, сырой SQL, транзакции.
  • Админка: ресурсы, инлайны, фильтры, действия.
  • Вью: FBV/CBV, middleware, контекст, get_object_or_404.
  • Безопасность: CSRF, XSS, настройки SECURE_*, ALLOWED_HOSTS.

Практика:

  • Мини‑блог с тэгами, админкой, поиском по полнотекстовому индексу (Postgres).

Критерии:

5) Django REST Framework (3–4 недели)

Цели: проектировать и реализовывать устойчивые REST‑API.

Что выучить:

  • Сериализаторы: Serializer/ModelSerializer, валидаторы уровня поля/объекта, вложенные сериализаторы, source, read_only/write_only.
  • Представления: APIView, GenericAPIView, ViewSet, ModelViewSet, mixins.
  • Роутеры и URL, namespacing, версионирование.
  • Пагинация (Limit/Offset, PageNumber), фильтры (django-filter), сортировки, поиск.
  • Права/пермишены, кастомные пермишены, throttling/ratelimit.
  • Аутентификация: JWT (djangorestframework-simplejwt), refresh/rotate, blacklist.
  • Ошибки: единый формат ошибок, кастомный ExceptionHandler.
  • Документация: OpenAPI/Swagger с drf-spectacular, схемы, примеры.
  • Тесты: API‑тесты (pytest‑django), фабрики (factory_boy), фикстуры БД.

Практика (проект):

  • Kanban API: User/Board/List/Card/Comment, роли (owner/member/guest), фильтры/поиск, пагинация, JWT, Swagger, 70%+ coverage.

Критерии:

6) PostgreSQL и данные (1–2 недели, параллельно)

Цели: проектировать схемы и писать эффективные запросы.

Что выучить:

  • Проектирование БД: нормализация (1–3НФ), денормализация, индексы (B‑tree, partial, composite), внешние ключи.
  • Транзакции и уровни изоляции, блокировки, deadlocks.
  • План выполнения: EXPLAIN/ANALYZE, оценка стоимости.
  • Миграции/сид‑данные, миграции с простой «катастрофоустойчивостью».

Практика:

  • Индексируйте горячие запросы Kanban API, сравните планы до/после.

Критерии:

7) Инфраструктура, безопасность и эксплуатация (2 недели)

Цели: собрать прод‑окружение и доставлять изменения безопасно.

Что выучить:

  • Docker/Compose: multi‑container (db, redis, backend, frontend), healthchecks, volume’ы, сети.
  • Настройка prod: Gunicorn + Nginx, статика/медиа, whitenoise (если нужно), логирование (JSON logs).
  • Переменные окружения и секреты, 12‑Factor, конфигурация для dev/stage/prod.
  • CI/CD: GitHub Actions (линты → тесты → build → deploy), кеширование зависимостей.
  • Наблюдаемость: Sentry, health‑checks, метрики (Prometheus/Grafana — по желанию).
  • Безопасность: CORS, rate limiting, заголовки безопасности, пароли/хеши, регулярные бэкапы БД.

Практика:

  • Docker Compose для всего стека, деплой backend (Render/Fly/Railway) + frontend (Vercel/Netlify), домен + HTTPS.

Критерии:

8) Склейка фронта и бэка (1–2 недели)

Цели: надёжный контракт и UX‑устойчивость.

Что выучить:

  • Соглашение о схемах: единый формат ошибок, поля next/previous/count/results (или аналог), коды статусов.
  • Аутентификация на фронте: хранение токенов (in‑memory + refresh), обновление/ревокация, guard routes.
  • RTK Query/React Query: кеш‑ключи, invalidation, optimistic updates, polling/websocket streaming.
  • Uploads: multipart на фронте, валидация размера/типа, предпросмотр; на бэке — валидация и S3‑совместимое хранилище.

Практика:

  • Подключите SPA к Kanban API, реализуйте роли и разграничение маршрутов, загрузку файлов.
